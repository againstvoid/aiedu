# 儿童生字记录 App 需求文档

## 1. App 基本信息

App 名称：
- 暂定：生字小本本

目标平台：
- Android

开发技术：
- Flutter

目标用户：
- 儿童
- 家长
- 老师

核心目标：
- 帮助儿童记录、查询和复习学习过的生字
- 支持标注生字是否已经学会
- 通过儿童友好的界面降低使用门槛


## 2. 核心功能概述

### 2.1 录入生字

用户可以手动录入一个或多个生字。

需要支持的信息：
- 生字
- 汉字图片
- 拼音，支持多音字
- 基本解释，支持按读音分别保存
- 备注
- 是否已学会

录入方式：
- 手动输入
- 用户输入生字后，自动从汉典页面抓取汉字图片、拼音、基本解释
- 如果是多音字，需要抓取多个读音以及每个读音对应的基本解释
- 自动抓取失败时，允许用户手动填写拼音和基本解释

校验规则：
- 生字不能为空
- 第一版限制只输入单个汉字
- 是否允许重复录入同一个生字：不允许 并 提醒用户


### 2.2 查询历史生字

用户可以查看已经录入过的所有生字。

支持使用拼音首字母或者输入生字过滤。

列表中展示的信息：
- 生字
- 拼音，多个读音用空格或顿号分隔
- 是否已学会
- 录入时间

列表排序方式：
- 按拼音排序
- 按录入时间倒序
- 按录入时间正序
- 其他：

列表右侧有字母表可以快速滑动，可以按照拼音首字母快速定位的指定生字范围。


### 2.3 标注是否已学会

用户可以对每个生字标记学习状态。

学习状态：
- 未学会
- 已学会

交互方式：
- 在生字详情页切换
- 在生字列表中直接切换
- 其他：


### 2.4 按学习状态过滤

用户可以按照学习状态筛选生字列表。

过滤条件：
- 全部
- 未学会
- 已学会

默认过滤条件：
- 全部 / 未学会 / 已学会

全局过滤：
- 配置页提供“隐藏已学会生字”开关
- 开启后，首页列表、搜索结果、详情页左右滑动的上一个/下一个范围都默认排除已学会生字
- 关闭后，按页面内筛选条件显示全部、未学会或已学会


## 3. 页面需求

### 3.1 首页 / 生字列表页

页面目标：
- 展示所有已录入的生字
- 支持快速查看学习状态
- 支持筛选已学会或未学会

页面元素：
- 顶部标题
- 生字搜索框
- 学习状态筛选控件
- 生字列表
- 新增生字按钮
- 配置入口

列表项展示：
- 生字
- 汉字图片
- 拼音，支持显示多个读音
- 简短基本解释
- 是否已学会
- 录入时间

用户操作：
- 点击生字进入详情页
- 点击新增按钮进入录入页
- 切换是否已学会
- 按状态过滤
- 进入配置页修改全局过滤设置


### 3.2 新增生字页

页面目标：
- 录入一个新的生字及相关信息

表单字段：
- 生字
- 汉字图片
- 拼音，支持多个读音
- 基本解释，支持按读音分别编辑
- 备注
- 是否已学会

用户操作：
- 保存
- 取消
- 输入生字后自动查询汉典页面并填充汉字图片、拼音、基本解释


### 3.3 生字详情页

页面目标：
- 查看单个生字的完整信息
- 修改学习状态
- 编辑或删除该生字

展示内容：
- 生字
- 汉字图片
- 拼音，支持多个读音
- 基本解释，支持按读音分组展示
- 备注
- 是否已学会
- 录入时间
- 最近修改时间

用户操作：
- 标记为已学会 / 未学会
- 编辑
- 删除
- 支持左右滑动切换到上一个或下一个生字

滑动规则：
- 详情页按照 `primary_pinyin` 拼音排序确定上一个和下一个生字
- 拼音相同时，按 `character` 或 `created_at` 作为稳定排序补充
- 如果配置页开启“隐藏已学会生字”，左右滑动时跳过已学会生字
- 滑动到边界时显示轻量提示，不循环跳转


### 3.4 编辑生字页

页面目标：
- 修改已经录入的生字信息

表单字段：
- 与新增生字页一致

用户操作：
- 保存修改
- 取消


### 3.5 配置页

页面目标：
- 管理 App 全局显示偏好

页面元素：
- 隐藏已学会生字开关

用户操作：
- 开启或关闭“隐藏已学会生字”
- 修改后立即影响首页列表、搜索结果和详情页左右滑动范围


## 4. 数据存储需求

数据存储方式：
- 本地存储
- 使用 SQLite 数据库存储
- 是否需要后端同步：否

SQLite 表结构建议：

### words 表

用于保存每个生字的基础信息。

字段：
- `id`：TEXT，主键，唯一 ID
- `character`：TEXT，生字，第一版限制单个汉字
- `glyph_image_url`：TEXT，汉字图片 URL
- `primary_pinyin`：TEXT，默认读音，用于列表展示和排序
- `note`：TEXT，用户备注
- `is_learned`：INTEGER，是否已学会，0 表示未学会，1 表示已学会
- `created_at`：TEXT，创建时间，ISO 8601 字符串
- `updated_at`：TEXT，更新时间，ISO 8601 字符串

索引：
- `idx_words_character`：基于 `character`，用于按汉字查询和重复检测
- `idx_words_is_learned`：基于 `is_learned`，用于按学习状态过滤
- `idx_words_created_at`：基于 `created_at`，用于按录入时间排序
- `idx_words_primary_pinyin`：基于 `primary_pinyin`，用于按拼音排序

约束：
- `character` 第一版建议唯一，避免重复录入同一个生字

### pronunciations 表

用于保存每个生字的一个或多个读音及对应基本解释，支持多音字。

字段：
- `id`：TEXT，主键，唯一 ID
- `word_id`：TEXT，关联 `words.id`
- `pinyin`：TEXT，拼音
- `zhuyin`：TEXT，注音，可为空
- `basic_explanation`：TEXT，该读音对应的基本解释
- `sort_order`：INTEGER，读音展示顺序

索引：
- `idx_pronunciations_word_id`：基于 `word_id`，用于查询某个生字的全部读音
- `idx_pronunciations_pinyin`：基于 `pinyin`，用于后续按拼音搜索或排序

约束：
- `word_id` 外键关联 `words.id`
- 删除生字时，级联删除对应读音记录

### settings 表

用于保存 App 全局配置。

字段：
- `key`：TEXT，主键，配置项名称
- `value`：TEXT，配置项值
- `updated_at`：TEXT，更新时间，ISO 8601 字符串

初始配置：
- `hide_learned_words`：`false`，是否全局隐藏已学会生字

索引：
- `sqlite_autoindex_settings_1`：由 `key` 主键自动创建，用于读取和更新配置

本地数据字段建议：

```json
{
  "id": "唯一ID",
  "character": "地",
  "glyphImageUrl": "https://img.zdic.net/kai/cn/5730.svg",
  "pronunciations": [
    {
      "pinyin": "dì",
      "zhuyin": "ㄉㄧˋ",
      "basicExplanation": "人类生长活动的所在；地球或地球的某部分。"
    },
    {
      "pinyin": "de",
      "zhuyin": "ㄉㄜ˙",
      "basicExplanation": "结构助词，用在词或词组之后表示修饰后面的谓语。"
    }
  ],
  "primaryPinyin": "dì",
  "note": "容易和字形相近的字混淆",
  "isLearned": false,
  "createdAt": "2026-01-01T10:00:00",
  "updatedAt": "2026-01-01T10:00:00"
}
```


## 5. 汉字信息自动抓取需求

数据来源：
- 汉典网页：`https://zdic.net/hans/{汉字}`
- 第一版通过解析网页 HTML 获取数据，不接入独立开放 API

自动抓取字段：
- 汉字图片
- 拼音，支持多音字
- 基本解释，支持按读音分组

字段来源说明：
- 汉字图片：解析页面中的字形图片地址，例如 `https://img.zdic.net/kai/cn/{Unicode}.svg`
- 拼音：解析页面顶部拼音文本，可能包含多个读音，例如“地”包含 `dì`、`de`
- 基本解释：解析页面“基本解释”区域内容；如果存在多个 `●字 拼音 注音` 分段，需要分别保存每个读音下的解释

触发时机：
- 在新增生字页输入单个汉字后自动查询
- 如果用户修改生字，重新查询并更新自动填充字段

保存策略：
- 自动抓取到的数据随生字记录一起保存到本地数据库
- 汉字图片第一版保存图片 URL，不强制下载图片文件到本地
- 多音字用 `pronunciations` 数组保存，每个元素包含 `pinyin`、`zhuyin`、`basicExplanation`
- `primaryPinyin` 用于列表排序和默认展示，默认取第一个读音，用户可手动调整
- 用户可以在保存前手动修改拼音和基本解释

网络异常处理：
- 查询失败时允许手动填写
- 显示友好提示
- 不影响本地保存

注意事项：
- 汉典网页不是正式开放 API，页面结构变化可能导致解析失效
- 如果后续发布或商用，需要确认汉典内容、图片和页面数据的使用许可
